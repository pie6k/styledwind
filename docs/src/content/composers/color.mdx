# Color Composer

The Color Composer provides a powerful API for managing colors in your components. It's designed to encourage creating reusable color systems that maintain consistency and accessibility across your application.

## Basic Usage

```tsx
import styled from "styled-components";
import { color } from "stylings";

const Button = styled.button`
  ${color({ color: "#007AFF" }).asBg.interactive};
`;
```

## Creating Reusable Color Systems

Start by defining your color palette and common color patterns:

```tsx
// Define base colors
const colors = {
  primary: color({ color: "#007AFF" }),
  secondary: color({ color: "#5856D6" }),
  success: color({ color: "#34C759" }),
  warning: color({ color: "#FF9500" }),
  error: color({ color: "#FF3B30" }),
  neutral: color({ color: "#8E8E93" }),
};

// Compose into semantic color patterns
export const semantic = {
  // Interactive elements
  button: colors.primary.asBg.interactive,
  link: colors.primary.asColor.interactive,
  
  // Status indicators
  successText: colors.success.asColor,
  warningText: colors.warning.asColor,
  errorText: colors.error.asColor,
  
  // Backgrounds
  cardBg: colors.neutral.asBg.secondary,
  modalBg: colors.neutral.asBg.tertiary,
  
  // Borders
  border: colors.neutral.asBorder.secondary,
};
```

## Color Definition

Define colors with various properties:

```tsx
// Basic color definition
${color({ color: "#007AFF" })}

// With custom foreground color
${color({ color: "#007AFF", foreground: "#FFFFFF" })}

// With custom hover and active states
${color({ 
  color: "#007AFF",
  hover: "#0056B3",
  active: "#004494"
})}

// With custom hover multiplier
${color({ 
  color: "#007AFF",
  hoverMultiplier: 1.5
})}
```

## Output Types

Specify how the color should be applied:

```tsx
// As background color
${color({ color: "#007AFF" }).asBg}

// As text color
${color({ color: "#007AFF" }).asColor}

// As border color
${color({ color: "#007AFF" }).asBorder}

// As outline color
${color({ color: "#007AFF" }).asOutline}

// As SVG fill color
${color({ color: "#007AFF" }).asFill}
```

## Opacity Variants

Create different opacity variants of a color:

```tsx
// Secondary opacity (55%)
${color({ color: "#007AFF" }).secondary}

// Tertiary opacity (30%)
${color({ color: "#007AFF" }).tertiary}

// Fully transparent
${color({ color: "#007AFF" }).transparent}

// Custom opacity
${color({ color: "#007AFF" }).opacity(0.5)}
```

## Interactive States

Add hover and active states automatically:

```tsx
// Add hover and active states
${color({ color: "#007AFF" }).interactive}

// Add hover state only
${color({ color: "#007AFF" }).hover}

// Add active state only
${color({ color: "#007AFF" }).active}

// Add muted state
${color({ color: "#007AFF" }).muted}

// Custom highlight ratio
${color({ color: "#007AFF" }).highlight(1.5)}
```

## Border Support

Add borders to background colors:

```tsx
// Add a border with the color
${color({ color: "#007AFF" }).withBorder}
```

## Foreground Colors

Automatically calculate contrasting foreground colors:

```tsx
// Get contrasting foreground color
${color({ color: "#007AFF" }).foreground}
```

## Example Components

Here's a practical example of how to compose color styles in real components:

```tsx
// Define base colors
const colors = {
  primary: color({ color: "#007AFF" }),
  neutral: color({ color: "#8E8E93" }),
};

// Compose into semantic patterns
const semantic = {
  button: colors.primary.asBg.interactive,
  card: colors.neutral.asBg.secondary.withBorder,
  text: colors.neutral.asColor,
  link: colors.primary.asColor.interactive,
};

// Use in components
const Button = styled.button`
  ${semantic.button};
`;

const Card = styled.div`
  ${semantic.card};
`;

const Text = styled.p`
  ${semantic.text};
`;

const Link = styled.a`
  ${semantic.link};
`;

const Icon = styled.svg`
  ${colors.primary.asFill};
`;
```

The Color Composer makes it easy to create a consistent color system across your application. By defining base colors and composing them into semantic patterns, you can maintain consistency while ensuring good contrast and accessibility.
