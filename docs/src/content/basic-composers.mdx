
# Basic Composers

There are several built-in composers `stylings` provides.

Note that all composers are infinitely chainable. In the later chapters, we will learn how to create our own composers.

For full list of composers and their capabilities, see [API](./api.mdx).


## `$flex`

```tsx
import { $flex } from 'stylings';

$flex.horizontal.alignCenter.gap(2);
$flex.vertical.center.wrap.inline;
// etc.
```

## `$grid`

```tsx
import { $grid } from 'stylings';

$grid.columns(2).rows(2).gap(2);
```

## `$common`

```tsx
import { $common } from 'stylings';

$common.opacity(0.5).absolute.inset(0).notSelectable;
$common.pt(10).pb(10).px(20);
$common.cursor('pointer');
$common.userSelect('none');
$common.pointerEvents('none');
$common.z(10);
$common.widthFull.heightFull;
$common.relative;
$common.overflowHidden;
$common.aspectRatio(16/9);

```

## `$animation`

```tsx
import { $animation } from 'stylings';

/**
 * We can first define some base animation style such as duration and easing function.
 * Then we can compose it into more specific animations.
 */
const $quickEasedAnimation = $animation.duration(200).easeInOut;

const animations = {
  fadeIn: $quickEasedAnimation.property('opacity', [0, 1]),
  fadeOut: $quickEasedAnimation.property('opacity', [1, 0]),
  slideUp: $quickEasedAnimation.property('y', [0, -10]),
  slideDown: $quickEasedAnimation.property('y', [0, 10]),
  spin: $animation.property('rotateZ', [0, 360]).linear.infinite,
}
```

## `$font`

```tsx filename="styles.ts"
import { $font } from 'stylings';

/**
 * Define a base font style with properties that are common to text styles we will use.
 * Then we can compose it into more specific fonts.
 */
const $baseFont = $font.family("Inter, sans-serif").size("1rem").lineHeight(1.5).antialiased;

const typo = {
  $copy: $baseFont,
  // We can override properties from the base font if needed
  $heading: $baseFont.size("2rem").lineHeight(1.25).bold, 
  $label: $baseFont.size("0.875rem"),
}
```

As composer is chainable, we can continue to compose them into even more specific styles.

```tsx filename="SmallLink.tsx"
import { typo } from './styles';
import { UI } from 'stylings';

function SmallLink() {
  return (
    <UI.a styles={[typo.$label.underline]}>
      Click me
    </UI.a>
  )
}
```

## `$color`

Color composer is somewhat opinionated. 

Each color in `stylings` is defined together with 3 variations of it:
- different states (hover, active). 
- "readable text" color (aka if given color is used as background, what color should be used for text to be readable).

All variations are optional. If not provided, `stylings` will automatically generate them using some heuristics.

```tsx filename="styles.ts"
import { $color } from 'stylings';

const danger = $color("#f00")
```



## `$surface`

The concept of `surface` is also somewhat opinionated.

In many apps, multiple components and controls share some (or all) characteristics.

For example, we might have buttons, selects and inputs that share size, padding, border radius, etc.

This is exactly what `surface` is for.
